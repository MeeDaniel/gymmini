# Техническое Задание

*Этот файл был скормлен Gemini 3.1 Pro (High)*

Gymmini: приложение для записей с тренировок. Помогает записывать информацию об упражнениях: что делал, сколько подходов, сколько повторений (или любая другая метрика).

## База данных

База данных должна хранить пользователей, их упражнения, записи тренировок, записи упражнений на тренировках.

1. **user**
  - `id`: `key_int`
  - `telegram_alias`: `bounded_string` (учти максимальную длину телеграм алиаса)

2. **workout**
  - `id`: `key_int`
  - `brief`: `bounded_string` *(Ультра краткое описание тренировки, например "День ног" или "На выносливость спины")*
  - `description`: `unbounded_string` (optionally) *(Подробное описание тренировки)*
  - `user_id`: `key_int` -> user

3. **exercise**
  - `id`: `key_int`
  - `name`: `bounded_string`
  - `description`: `unbounded_string` (or very huge, optionally) *(как делать упражнение, может какие-то записи)*
  - `image`: `image? image_link?` (optional) *(не знаю как будет лучше. Возмозжно хранить изображеиня будет слишком затратно по месту. В тг вроде можно пользоваться хранилищем телеграма, но тг не факт, что будет единственным интерфейсом. Возможно картинки будут сохранятся через веб сайт)*
  - `user_id`: `key_int` -> user

4. **workout_note**
  - `id`: `key_int`
  - `brief`: `bounded_string` *(Ультра краткое описание тренировки, например "День ног" или "На выносливость спины")*
  - `description`: `unbounded_string` (optionally) *(Подробное описание тренировки)*
  - `date`: `date`
  - `time`: `string` (optionally)
  - `user_id`: `key_int` -> user

5. **exercise_note**
  - `id`: `key_int`
  - `exercise_id`: `key_int` -> exercise
  - `workout_id`: `key_int` -> workout
  - `notes`: `unbounded_string`

6. **set**
  - `id`: `key_int`
  - `exercise_note_id`: `key_int` -> exercise_note
  - `reps`: `int | null`
  - `weight`: `float | null` *(все float значения принимать как с точкой, так и с запятой, так и вообще без разделительного знака. Возвращать с точкой, если необходимо. Т.е. принимаются "1.1", "1,1" и "1", а возвращается "1" или "1.1")*
  - `duration`: `float | null`
  - `distance`: `float | null`

Описание работы:
1. Пользователь создаёт аккаунт с привязанным телеграмом (изначально приложение будет работать только через телеграм, потом, возможно, будут подвязаны другие интерфейсы).
2. Далее он может описать свои тренировки и упражнения.
3. У тренировок есть конкретная цель
4. Упражнения описыавют что надо делать
5. Во время тренировки пользователь создаёт `workout_note` - берётся `workout` и копируется с заданными датой и временем (время необязательно, нужно на случай, если тренировок в день несколько. Описывается просто строкой, т.е для удобство можно "вечер" или более точно "18:00")
6. Во время тренировки пользователь добавляет в тренировку `exercise_note` - тоже копия `exercise`
7. Во время выполнения упражнения пользователь добавляет в тренировку информацию о своих подходах, создавая `set`

## Архитектура

Требуется собрать приложение с каким-нибудь бекендом, который можно будет спокойно запустить на любом сервере. Поверхностно бекенд представляет собой api. Фронтенд возможно будет меняться. Пока что это telegram_bot, но далеко не факт, что будет только он.

UPD: используй FAST_API и Sqlite, для хостинга лучше без докера, бота через aiogram. Бота автоматизировать через онлайн кнопки, не забывать про кнопки назад и в главное меню.

## Прочие пожелания

Здесь указаны разные мысли о будущем проекта. Разрабатывать их не надо, можно просто подготовить проект к этому

В будущем планируется создание разных паков тренировок, которыми люди смогут делиться. Поделиться будет означать скопировать тренировки и упражнения из пака на чей-то аккаунт. Также, возможно, будут разные роли у пользователей (не нужно добавлять это в базу данных и api). Это тренер, ученик и простой пользователь. Тренер может создавать тренировки и контралировать записи своих учеников.
