# Архитектура проекта Gymmini

## Общее описание
Gymmini — это монолитное приложение для личного трекинга тренировок. Оно объединяет в себе Telegram-бота (через `aiogram`) и веб-сервер (через `FastAPI`). Вся система запускается в рамках одного процесса (`main.py`) с помощью `asyncio.gather`. 

Такая архитектура выбрана для максимального упрощения развертывания проекта на недорогих серверах без необходимости настраивать сложные CI/CD пайплайны и оркестраторы вроде Docker (хотя добавление Docker возможно в будущем).

## Стек технологий
- **Управление зависимостями**: `uv` (самый быстрый менеджер на данный момент).
- **БД**: `SQLite` (с асинхронным драйвером `aiosqlite`). База хранится локально в файле `gymmini.sqlite3`.
- **ORM и миграции**: `SQLAlchemy 2.0` (async) и `Alembic`.
- **Бот**: `aiogram 3.x`.
- **API**: `FastAPI` + `Uvicorn`.
- **Тесты**: `pytest` + `pytest-asyncio`.
- **Конфигурация**: `pydantic-settings` (читает `.env` файл).

## Структура проекта
```text
Gymmini/
├── src/
│   ├── api/          # FastAPI роутеры (в будущем для web-фронта)
│   ├── bot/          # Логика aiogram (хендлеры, клавиатуры, состояния)
│   │   ├── messages.py # Централизованное хранилище всех текстов бота
│   │   └── handlers.py # Обработчики команд и сообщений
│   ├── core/         # Конфигурация (config.py на базе pydantic-settings)
│   ├── db/           # Подключение к БД, базовая модель (base.py)
│   ├── models/       # SQLAlchemy модели (user.py, workout.py, exercise.py)
│   └── services/     # Бизнес-логика (CRUD), независимая от интерфейса
├── tests/            # Тесты (pytest)
├── media/            # Локальное хранилище для пользовательских картинок
├── docs/             # Документация по проекту
├── alembic/          # Миграции БД
├── main.py           # Точка входа в приложение
└── .env              # Переменные окружения (секреты)
```

## База данных (Сущности)

1. **User** (`users`): Пользователь. Имеет `telegram_id` и `telegram_alias`.
2. **Workout** (`workouts`): *Шаблон* тренировки. Содержит базовое описание (например, "День ног").
3. **Exercise** (`exercises`): *Шаблон* упражнения. Содержит описание и путь к картинке в `media/`.
4. **WorkoutNote** (`workout_notes`): *Проведенная* тренировка. Копируется из `Workout` при начале тренировки. Имеет дату и время. Привязана к `Workout` для ведения статистики.
5. **ExerciseNote** (`exercise_notes`): *Запланированное/выполненное* упражнение в рамках конкретной `WorkoutNote`.
6. **Set** (`sets`): Подход. Привязан к `ExerciseNote`. Содержит количество повторений, вес, время или дистанцию.

## Правила разработки

1. **Захардкоженные тексты**: ВСЕ текстовые сообщения для пользователя (особенно ответы бота) должны храниться в `src/bot/messages.py`. Запрещено хардкодить русский/английский текст напрямую в `handlers.py`.
2. **Бизнес-логика**: Обработчики бота (`handlers`) не должны содержать сложной логики. Они должны вызывать функции из папки `src/services/` (CRUD).
3. **Тестирование**: Все новые функции в `src/services/` должны покрываться тестами в `tests/`. Запуск тестов: `uv run pytest`.
4. **Добавление зависимостей**: Осуществляется только через `uv add <package>`.
